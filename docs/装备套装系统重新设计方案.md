# 传奇游戏引擎装备套装系统重新设计方案

## 设计概述

基于现有的套装系统，重新设计支持按套装数量生效不同功能的增强套装系统。保留现有的套装勾选列表功能，新增数量条件检测和分级效果系统。

## 现有系统分析

### 当前套装系统结构

1. **数据结构** (`TSetItems`)：
   ```pascal
   TSetItems = packed record
     sHint: string[50];                                          // 套装提示信息
     Items: array[Low(THumanUseItems)..High(THumanUseItems)] of string[14]; // 装备名称数组
     Value: array[0..30] of Word;                                // 属性加成值
     HideValue: Boolean;                                         // 是否隐藏数值
   end;
   ```

2. **检测逻辑**：
   - 通过 `g_SetItemsArr` 数组标记已装备的套装项目
   - 在 `RecalcAbilitys()` 中检测套装完整性
   - 全套装备才生效属性加成

3. **配置格式**：
   ```ini
   [套装名称]
   Items=装备1,装备2,装备3,...
   Values=属性类型.数值,属性类型.数值,...
   HideValue=0/1
   ```

### 现有系统限制

1. **只支持全套装效果** - 必须穿戴完整套装才生效
2. **单一效果层级** - 不支持根据数量生效不同效果
3. **配置不够灵活** - 无法定义多层级的属性加成

### 3. 检测逻辑实现

#### 套装数量检测函数
```pascal
function TPlayObject.CheckSuiteCount(nSuiteIndex: Integer): Integer;
var
  i: Integer;
  nCount: Integer;
  StdItem: pTStdItem;
  SetItem: pTEnhancedSetItems;
begin
  Result := 0;
  if (nSuiteIndex < 0) or (nSuiteIndex >= g_EnhancedSetItemsList.Count) then
    Exit;
    
  SetItem := g_EnhancedSetItemsList[nSuiteIndex];
  nCount := 0;
  
  // 检测每个装备位置
  for i := Low(THumanUseItems) to High(THumanUseItems) do begin
    if (m_UseItems[i].wIndex > 0) and (m_UseItems[i].Dura > 0) then begin
      StdItem := UserEngine.GetStdItem(m_UseItems[i].wIndex);
      if StdItem <> nil then begin
        // 检查是否是套装中的装备
        if SetItem.Items[i] <> '' then begin
          if CompareText(UserEngine.GetStdItemName(m_UseItems[i].wIndex), SetItem.Items[i]) = 0 then begin
            Inc(nCount);
          end;
        end;
      end;
    end;
  end;
  
  Result := nCount;
end;
```

#### 套装效果计算函数
```pascal
procedure TPlayObject.CalcSuiteEffects;
var
  i, k, nCount: Integer;
  SetItem: pTEnhancedSetItems;
  SuiteStatus: pTPlayerSuiteStatus;
  nActiveLevel: Integer;
begin
  // 重置套装状态
  FillChar(m_SuitesStatus, SizeOf(m_SuitesStatus), 0);
  
  // 检测每个套装
  for i := 0 to g_EnhancedSetItemsList.Count - 1 do begin
    SetItem := g_EnhancedSetItemsList[i];
    SuiteStatus := @m_SuitesStatus[i];
    
    SuiteStatus.nSuiteIndex := i;
    SuiteStatus.nEquippedCount := CheckSuiteCount(i);
    
    if SetItem.boUseCountEffect then begin
      // 启用数量效果模式
      nActiveLevel := GetActiveSuiteLevel(SetItem, SuiteStatus.nEquippedCount);
      SuiteStatus.nActiveEffectLevel := nActiveLevel;
      
      // 应用对应等级的效果
      if nActiveLevel > 0 then begin
        ApplySuiteCountEffect(SetItem, nActiveLevel);
      end;
    end else begin
      // 传统全套装模式
      if SuiteStatus.nEquippedCount = SetItem.nTotalCount then begin
        SuiteStatus.boFullSuiteActive := True;
        ApplyFullSuiteEffect(SetItem);
      end;
    end;
  end;
end;
```

#### 获取生效等级函数
```pascal
function TPlayObject.GetActiveSuiteLevel(SetItem: pTEnhancedSetItems; nCount: Integer): Integer;
var
  i: Integer;
  nMaxLevel: Integer;
begin
  Result := 0;
  nMaxLevel := 0;
  
  // 从高到低检查哪个等级可以生效
  for i := High(SetItem.CountEffects) downto Low(SetItem.CountEffects) do begin
    if SetItem.CountEffects[i].boEnabled and (nCount >= SetItem.CountEffects[i].nRequiredCount) then begin
      if SetItem.CountEffects[i].nRequiredCount > nMaxLevel then begin
        nMaxLevel := SetItem.CountEffects[i].nRequiredCount;
        Result := i;
      end;
    end;
  end;
end;
```

#### 应用套装效果函数
```pascal
procedure TPlayObject.ApplySuiteCountEffect(SetItem: pTEnhancedSetItems; nLevel: Integer);
var
  i: Integer;
  Effect: pTSuiteCountEffect;
begin
  if (nLevel < Low(SetItem.CountEffects)) or (nLevel > High(SetItem.CountEffects)) then
    Exit;
    
  Effect := @SetItem.CountEffects[nLevel];
  if not Effect.boEnabled then
    Exit;
    
  // 应用属性加成
  for i := Low(Effect.Value) to High(Effect.Value) do begin
    if Effect.Value[i] > 0 then begin
      ApplySuiteAttribute(i, Effect.Value[i]);
    end;
  end;
end;
```

### 5. 客户端显示

#### 套装信息消息格式
```pascal
// 服务端发送套装状态消息
procedure TPlayObject.SendSuiteStatus;
var
  i, k: Integer;
  sSendMsg, sStr: string;
  SetItem: pTEnhancedSetItems;
  SuiteStatus: pTPlayerSuiteStatus;
begin
  sSendMsg := '';
  
  for i := 0 to g_EnhancedSetItemsList.Count - 1 do begin
    SetItem := g_EnhancedSetItemsList[i];
    SuiteStatus := @m_SuitesStatus[i];
    
    if SuiteStatus.nEquippedCount > 0 then begin
      sStr := IntToStr(i) + '|' +                           // 套装索引
              IntToStr(SuiteStatus.nEquippedCount) + '|' +   // 已装备数量
              IntToStr(SetItem.nTotalCount) + '|' +          // 总数量
              IntToStr(Byte(SetItem.boUseCountEffect)) + '|' + // 是否启用数量效果
              IntToStr(SuiteStatus.nActiveEffectLevel) + '|' + // 当前生效等级
              SetItem.sHint + '|';                           // 套装名称
              
      // 添加效果描述
      if SetItem.boUseCountEffect and (SuiteStatus.nActiveEffectLevel >= 0) then begin
        sStr := sStr + SetItem.CountEffects[SuiteStatus.nActiveEffectLevel].sEffectHint;
      end;
      
      sSendMsg := sSendMsg + EncodeString(sStr) + '/';
    end;
  end;
  
  if sSendMsg <> '' then begin
    m_DefMsg := MakeDefaultMsg(SM_SUITE_STATUS, 0, 0, 0, 0);
    SendSocket(@m_DefMsg, sSendMsg);
  end;
end;
```

### 6. 兼容性保证

#### 向后兼容处理
```pascal
procedure LoadEnhancedSetItems;
var
  i: Integer;
  OldSetItem: pTSetItems;
  NewSetItem: pTEnhancedSetItems;
begin
  // 将现有套装转换为增强套装格式
  for i := 0 to g_SetItemsList.Count - 1 do begin
    OldSetItem := g_SetItemsList[i];
    New(NewSetItem);
    
    // 复制基础信息
    NewSetItem.sHint := OldSetItem.sHint;
    NewSetItem.Items := OldSetItem.Items;
    NewSetItem.Value := OldSetItem.Value;
    NewSetItem.HideValue := OldSetItem.HideValue;
    
    // 设置默认值
    NewSetItem.nTotalCount := CountSuiteItems(OldSetItem);
    NewSetItem.boUseCountEffect := False; // 默认不启用数量效果
    
    // 清空数量效果
    FillChar(NewSetItem.CountEffects, SizeOf(NewSetItem.CountEffects), 0);
    
    g_EnhancedSetItemsList.Add(NewSetItem);
  end;
end;
```

### 7. 配置管理

#### 配置加载函数
```pascal
function LoadSuiteCountEffect(SetItem: pTEnhancedSetItems; const sSection: string): Boolean;
var
  i: Integer;
  sKey, sValue, sValues, sHint: string;
  nCount: Integer;
begin
  Result := False;
  
  // 检查是否启用数量效果
  SetItem.boUseCountEffect := Conf.ReadBool(sSection, 'UseCountEffect', False);
  if not SetItem.boUseCountEffect then
    Exit;
    
  SetItem.nTotalCount := Conf.ReadInteger(sSection, 'TotalCount', 0);
  
  // 加载各个数量等级的效果
  for i := 2 to SetItem.nTotalCount do begin
    sKey := 'CountEffect' + IntToStr(i);
    sValue := Conf.ReadString(sSection, sKey, '');
    
    if sValue <> '' then begin
      // 解析配置: "1,生命值+100,攻击力+10"
      sValue := GetValidStr3(sValue, sCount, [',']);
      if StrToIntDef(sCount, 0) = 1 then begin
        SetItem.CountEffects[i div 2].boEnabled := True;
        SetItem.CountEffects[i div 2].nRequiredCount := i;
        SetItem.CountEffects[i div 2].sEffectHint := sValue;
        
        // 加载属性数值
        sKey := 'CountEffect' + IntToStr(i) + 'Values';
        sValues := Conf.ReadString(sSection, sKey, '');
        LoadSuiteValues(SetItem.CountEffects[i div 2].Value, sValues);
        
        Result := True;
      end;
    end;
  end;
end;
```

## 实现优势

### 1. 功能增强
- **渐进式效果** - 支持2/13、4/13、6/13、8/13、10/13、13/13等多层级效果（最多支持16个层级）
- **灵活配置** - 每个层级可以配置不同的属性加成
- **直观显示** - 客户端可以清晰显示当前套装状态和效果
- **完整装备支持** - 支持所有16个装备部位的套装组合

### 2. 兼容性
- **完全兼容** - 现有套装配置无需修改即可正常工作
- **平滑升级** - 可以逐步将套装升级为数量效果模式
- **配置简单** - 只需添加新的配置项即可启用新功能

### 3. 扩展性
- **多套装支持** - 同时支持多个套装的数量效果
- **属性扩展** - 可以轻松添加新的属性类型
- **效果叠加** - 支持不同套装效果的叠加

### 4. 用户体验
- **即时反馈** - 装备或卸下装备时立即显示效果变化
- **清晰提示** - 显示当前套装进度和下一层级效果
- **策略性** - 玩家可以根据需要选择不同的装备搭配
