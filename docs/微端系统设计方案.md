# 传奇游戏引擎微端系统设计方案

## 系统架构概述

微端系统采用分层架构设计，包含以下核心组件：

```
客户端 <---> 微端网关 <---> 微端服务器 <---> 资源文件系统
   |           |              |               |
   |           |              |               +-- 文件索引数据库
   |           |              |               +-- 资源文件存储
   |           |              |               +-- 补丁管理系统
   |           |              |
   |           |              +-- 资源请求处理
   |           |              +-- 文件传输管理
   |           |              +-- 缓存管理
   |           |
   |           +-- 客户端连接管理
   |           +-- 资源请求转发
   |           +-- 负载均衡
   |
   +-- 资源检测
   +-- 下载管理
   +-- 本地缓存
```

## 核心组件设计

### 1. 微端网关 (MicroGate)

**职责**：
- 处理客户端的资源请求
- 与微端服务器通信
- 负载均衡和连接管理
- 资源传输优化

**主要功能**：
- 客户端连接管理
- 资源请求路由
- 数据压缩传输
- 断点续传支持
- 并发下载控制

### 2. 微端服务器 (MicroServer)

**职责**：
- 维护完整的资源文件索引
- 处理资源查找和读取
- 管理补丁文件
- 提供高效的文件服务

**主要功能**：
- 文件索引管理
- 快速文件查找
- 补丁差分计算
- 文件完整性校验
- 缓存优化

### 3. 资源索引系统

**职责**：
- 建立完整的客户端文件索引
- 支持快速文件查找
- 管理文件版本和依赖关系
- 优化磁盘访问性能

  MC_DOWNLOAD_COMPLETE   = $F104;  // 下载完成

  // 微端网关到微端服务器
  MS_QUERY_RESOURCE      = $F201;  // 查询资源
  MS_GET_FILE_DATA       = $F202;  // 获取文件数据
  MS_CHECK_FILE_HASH     = $F203;  // 检查文件哈希
  
  // 微端服务器到微端网关
  MS_RESOURCE_FOUND      = $F301;  // 资源找到

## 文件索引系统设计

### 索引数据库结构

```sql
-- 文件索引表
CREATE TABLE FileIndex (
    FileId INTEGER PRIMARY KEY AUTOINCREMENT,
    FileName VARCHAR(255) NOT NULL,
    FilePath VARCHAR(500) NOT NULL,
    FileSize BIGINT NOT NULL,
    FileHash VARCHAR(32) NOT NULL,
    Version INTEGER NOT NULL,
    CreateTime DATETIME NOT NULL,
    UpdateTime DATETIME NOT NULL,
    CompressType TINYINT DEFAULT 0,
    CompressSize BIGINT DEFAULT 0,
    Dependencies TEXT,
    INDEX idx_filename (FileName),
    INDEX idx_hash (FileHash),
    INDEX idx_version (Version)
);

-- 补丁信息表
CREATE TABLE PatchInfo (
    PatchId INTEGER PRIMARY KEY AUTOINCREMENT,
    FromVersion INTEGER NOT NULL,
    ToVersion INTEGER NOT NULL,
    PatchFile VARCHAR(255) NOT NULL,
    PatchSize BIGINT NOT NULL,
    PatchHash VARCHAR(32) NOT NULL,
    CreateTime DATETIME NOT NULL,
    INDEX idx_version (FromVersion, ToVersion)
);

-- 下载统计表
CREATE TABLE DownloadStats (
    StatId INTEGER PRIMARY KEY AUTOINCREMENT,
    FileId INTEGER NOT NULL,
    DownloadCount INTEGER DEFAULT 0,
    TotalBytes BIGINT DEFAULT 0,
    LastAccess DATETIME,
    FOREIGN KEY (FileId) REFERENCES FileIndex(FileId)
);
```

## 性能优化策略

### 1. 缓存策略
- **内存缓存**：热点文件缓存在内存中
- **磁盘缓存**：最近访问的文件缓存到高速磁盘
- **分布式缓存**：多个微端服务器间的缓存同步

### 2. 传输优化
- **数据压缩**：支持多种压缩算法（LZ4、ZLIB、LZO）
- **分块传输**：大文件分块传输，支持并发下载
- **断点续传**：网络中断后可继续下载
- **增量更新**：只传输文件差异部分

### 3. 并发控制
- **连接池管理**：复用网络连接
- **限流控制**：防止服务器过载
- **优先级队列**：重要文件优先下载

## 安全机制

### 1. 文件完整性
- **哈希校验**：每个文件都有MD5/SHA256校验
- **数字签名**：关键文件使用数字签名验证
- **版本控制**：严格的版本管理防止回滚攻击

### 2. 传输安全
- **加密传输**：敏感数据使用AES加密
- **身份验证**：客户端身份验证
- **访问控制**：基于权限的资源访问控制

### 3. 防护措施
- **频率限制**：防止恶意大量请求
- **IP白名单**：限制访问来源
- **异常监控**：实时监控异常行为

**微端网关配置**：
- CPU: 4核心以上
- 内存: 8GB以上
- 网络: 千兆网卡
- 存储: SSD缓存盘

**微端服务器配置**：
- CPU: 8核心以上
- 内存: 16GB以上
- 网络: 千兆网卡
- 存储: 高速SSD + 大容量HDD

## 技术风险和应对

### 主要风险
1. **网络带宽瓶颈**：大量并发下载可能导致带宽不足
2. **存储IO瓶颈**：大量文件读取可能导致磁盘IO瓶颈
3. **内存使用过高**：文件缓存可能导致内存不足

### 应对措施
1. **带宽管理**：实施智能限流和负载均衡
2. **存储优化**：使用SSD缓存和异步IO
3. **内存管理**：实施LRU缓存策略和内存监控
